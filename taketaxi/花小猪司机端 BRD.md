## 执行摘要 (Executive Summary)

### 产品愿景
花小猪司机端致力于打造**最懂司机的赚钱工具**，以"听单更自由、收入更透明、奖励更丰厚"为核心价值主张，成为网约车司机首选的工作平台。

### 核心业务指标 (North Star Metrics)
| 指标 | 当前基线 | 目标值 | 衡量周期 |
|:-----|:--------:|:------:|:--------:|
| 司机日均在线时长 | - | ≥6 小时 | 日 |
| 司机月留存率 | - | ≥75% | 月 |
| 订单成交率 | - | ≥90% | 日 |
| 司机日均收入 | - | ≥300 元 | 日 |
| 乘客满意度 | - | ≥4.8/5.0 | 周 |

### 关键业务规则摘要
1. **安全合规**：真人检测 + 防疫检测每日 mandatory，累计服务 4 小时强制休息 20 分钟
2. **收入提现**：每周二/四 9:00-22:00，T-2 冻结期，单次 3-15000 元
3. **推荐奖励**：推荐乘客 2 元/人（普通）或 4 元/人（超级码），次日到账
4. **隐私保护**：虚拟中间号每单不同，订单支付 30 分钟后失效
5. **服务城市**：近 30 天 60% 订单在另一城市→每周一自动变更

### 需求优先级分布
| 优先级 | 数量 | 占比 | 说明 |
|:------:|:----:|:----:|:-----|
| P0 - 紧急重要 | 28 | 39% | 核心订单流程、安全合规、收入提现 |
| P1 - 重要不紧急 | 32 | 45% | 奖励体系、服务中心、设置管理 |
| P2 - 优化体验 | 11 | 16% | 消息通知、导航设置、关于页面 |

## 1. 文档概述

### 1.1 文档目的

本文档旨在详细描述花小猪打车 APP 司机端的业务需求，为产品设计、技术开发、测试验证提供统一的需求基线。文档基于现有 UI 设计稿（71 个页面）进行功能梳理和业务规则定义。

### 1.2 产品背景

花小猪打车是滴滴出行旗下的年轻化打车品牌，以"花小猪，打车更便宜"为核心定位。司机端 APP 是平台运力的核心载体，为司机提供订单接收、行程服务、收入管理等全套服务工具。

### 1.3 适用范围

本文档适用于花小猪打车司机端移动端应用（iOS/Android），涵盖司机从注册登录到日常接单、收入提现的全业务流程。

### 1.4 术语定义

| 术语 | 定义 |
|:----:|:-----|
| 听单 | 司机开启接收订单的状态 |
| 收车 | 司机结束当日运营，停止接收订单 |
| 成交率 | 司机接单后完成行程的比率 |
| 拒单 | 司机主动拒绝已分配的订单 |
| 保底收入 | 平台承诺的最低收入保障 |
| 真人检测 | 司机本人人脸识别验证 |
| 防疫检测 | 司机每日出车前的健康检查流程 |

## 2. 产品定位与业务目标

### 2.1 产品定位

花小猪司机端是网约车司机的一站式工作平台，帮助司机高效接单、合规运营、透明赚钱。

### 2.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|:--------:|:---------|:---------|
| 运力规模 | 吸引并留存高质量司机 | 日活司机数、周留存率 |
| 接单效率 | 提升司机接单转化率 | 听单时长、接单率、成交率 |
| 司机收入 | 让司机清晰了解收入，提升赚钱感知 | 日均收入、满意度评分 |
| 合规运营 | 确保司机资质合规、服务规范 | 真人检测通过率、防疫检测完成率 |
| 拉新增长 | 通过推荐机制扩大司机和乘客规模 | 推荐司机数、推荐乘客数 |

### 2.3 核心价值主张

- **对司机**：听单更自由、收入更透明、奖励更丰厚
- **对平台**：运力更稳定、服务更规范、增长更可持续

## 3. 用户角色与使用场景

### 3.1 用户角色定义

| 角色 | 描述 | 核心诉求 |
|:----:|:-----|:---------|
| 新注册司机 | 刚完成注册，尚未开始接单 | 快速熟悉流程、完成新手任务 |
| 活跃司机 | 每日稳定出车接单 | 多接单、多赚钱、奖励最大化 |
| 兼职司机 | 不定期出车，时间灵活 | 自由听单、灵活收车 |
| 全职司机 | 以网约车为主要职业 | 稳定收入、长期奖励、职业成长 |

### 3.2 核心使用场景

#### 场景 1：司机每日出车流程
```
登录 → 真人检测 → 防疫检测 → 开启听单 → 接收订单 → 完成行程 → 收车
```

#### 场景 2：司机接单服务流程
```
听单中 → 订单推送 → 确认接单 → 导航至上车点 → 到达确认 → 验证乘客 → 接到乘客 → 导航至目的地 → 到达确认 → 确认账单 → 行程结束
```

#### 场景 3：司机收入管理流程
```
查看今日收入 → 查看奖励进度 → 申请提现 → 到账确认
```

#### 场景 4：司机推荐拉新流程
```
进入推荐页面 → 查看奖励规则 → 生成邀请码/链接 → 分享给好友 → 好友完成注册/首单 → 获得奖励
```

#### 6.2.4 七日成交率

**功能描述：** 展示司机近 7 日成交率，反映接单后履约情况。

**页面元素：**
| 元素 | 类型 | 说明 |
|:-----|:----:|:-----|
| 页面标题 | 文本 | "七日成交率" |
| 日期选择器 | 下拉 | 默认为当天 |
| 成交率环 | 环形图表 | "100% 七日成交率"（粉色环） |
| 规则说明标题 | 文本 | "规则说明"（带粉色下划线） |
| 规则列表 | 有序列表 | 4 条详细计算规则 |

**计算规则：**
1. 七日成交率 = 近 7 日订单完成量（不含今日）/ 近 7 日接单量（不含今日）* 100%
2. 接单前拒绝的订单不降低成交率
3. 乘客取消的订单，也会降低成交率，建议接单后先联系乘客确认行程
4. 如司机存在作弊行为（包括但不限于使用作弊器、虚假交易等），将会对成交率产生影响，仿花小猪将取消参加奖励活动的资格

**业务规则：**
- 统计周期：近 7 日（滚动窗口），不含当天
- 成交率计算：完成量 / 接单量（含乘客取消）
- 拒单影响：接单前拒绝不计入分母；接单后拒绝计入分母但不计入分子
- 作弊惩罚：发现作弊行为可取消奖励资格
- 派单影响：成交率过低可能降低派单优先级

#### 6.2.5 规则说明（服务时长）

**功能描述：** 详细说明累计服务时长和连续休息时长规则，防疲劳驾驶。

**页面元素：**

|:-----|:----:|:-----|

**累计服务时长规则：**

- 累计服务时长满 4 小时，将强制收车 20 分钟

**连续休息时长规则：**
- 出车状态下的听单时间、收车时间均算做休息时间
- 连续休息满 20 分钟可清零累计服务时长

3. 累计服务时长为 2 小时，休息 10 分钟后出车服务 2 小时，因休息不满 20 分钟，所以累计服务时长为 2+2 小时，需要强制连续休息 20 分钟。

**业务规则：**
- 强制休息：满 4 小时必须休息 20 分钟才能继续听单
- 主动休息：累计未满 4 小时前主动休息 20 分钟可清零
- 休息不足：休息不满 20 分钟重新接单，休息时长清零，累计时长继续累加
- 安全合规：符合网约车驾驶员疲劳驾驶管理规定

### 11.6 核心业务规则汇总

**安全合规规则：**
- 真人检测：每日首次出车 Mandatory，3 次失败转人工
- 防疫检测：每日首次出车 Mandatory，体温>37.3°C 禁止出车
- 疲劳驾驶：累计服务 4 小时强制休息 20 分钟

**订单服务规则：**
- 手机号码验证：输入乘客手机号后 4 位，3 次错误需联系客服
- 到达判断：上车点<100 米、目的地<200 米才允许点击到达
- 成交率：近 7 日完成量/接单量（不含今日），乘客取消也降低成交率

**收入提现规则：**
- 提现时间：每周二、周四 9:00-22:00
- 可提现金额：T-2 天前收入（2 天冻结期）
- 提现限额：单次 3-15000 元，每周累计≤15000 元
- 到账时效：24 小时内

**推荐奖励规则：**
- 推荐乘客：2 元/人（普通扫码），4 元/人（车内超级码）
- 奖励发放：次日到账
- 防作弊：设备、账户、手机号、微信、支付宝、银行卡任一相同视为同一用户

**隐私安全规则：**
- 隐私号码：虚拟中间号每个订单不同，支付 30 分钟后失效
- 紧急联系人：仅在使用 110 报警功能后通知
- 录音保护：行程期间全程录音，仅用于安全和纠纷场景

**服务城市变更规则：**
- 自动变更：近 30 天 60% 以上订单在另一城市完成
- 更新时间：每周一自动更新
